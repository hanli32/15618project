Parallel Computing Distance Based on Heat Flow

Team Members: Bo Feng, Han Li

Summary

We are going to parallel computing the geodesic distance to a specified subset of a given domain (grids, triangle meshes, point clouds) on GPU.

Background

Traditional geodesics distance computing faces challenge of generalization and efficiency. The heat method solves the problem by making use of standard linear equations, which proves to be fast, general and huge potential for parallelization. In practice, distance is updated an order of magnitude with heat method faster than with state-of-the-art methods, while maintaining a comparable level of accuracy. The method requires only standard differential operators and can hence be applied on a wide variety of domains (grids, triangle meshes, point clouds, etc.).

However, None of the current implementations of the heat method fully exploit its potential for speed. Being attracted to this creative approach for distance computing, we are highly motivated to put our effort and apply parallel computing techniques from 15618 course to implementing this idea. That is, our goal is to produce a high-performance, general-purpose and user-friendly library of geodesic distance computation, which supports multiple domains, including triangle meshes, grids and point clouds and fully exploit parallel computing power of GPU-based system.

The Challenge

There are great challenges lying in this project since it involves complex graphics/geometry knowledge and non-trivial parallelism for implementation on GPU-based system.

Despite interest in this topic, neither of us has sufficient experience in graphic fields. Therefore, it might not be easy for us to fast understand the model and framework in the project. And this would certainly increase difficulty for us to exploit the potential parallelism during writing code for implementation and further optimization.

Furthermore, GPU-based system has a distinction between the device memory and host memory. At this moment, itâ€™s not obvious to us how to efficiently organize the workload so as to reduce the traffic overhead incurred by moving data back and forth.

Resources

Generally, we will start from scratch on this project. However, the idea is mainly from the original heat method paper, and we can also refer to the existing library written in ANSI-C as a good sequential implementation example for triangulated surfaces.

Goals and Deliverables

Since the starter code gives triangle meshes implementation, we hope to implement this algorithm on other domains like grids and point clouds.

We plan to parallel this implementation with CUDA. We do not have a precise performance goal now but we will set an aim in later phases after having a better understand of this algorithm.

We plan to demonstrate our project by given two points then compute the geodesic distance in real time.

Platform Choice

We are going to use CUDA framework on both GHC machines and latedays clusters in order to have a low-level control of GPUs and maximize parallelism speedup.
